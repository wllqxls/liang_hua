# 渐进式智能参数搜索设计

## 目标

把当前需要反复手动更换参数的搜索方式改成单币种、一键式渐进搜索。在普通本机上以 5–10 分钟为时间预算，自动完成广泛粗筛、局部精搜和稳健性验证。

搜索目标是找到可重复的策略结构，不是找到某一段行情中收益最高的单组参数。

## 用户操作

用户保留这些输入：

- 交易对象：一次只搜索一个币种。
- 初始资金和单笔逐仓金额：作为风险约束，不参与搜索。
- 手续费、滑点、资金费率和维持保证金率：作为固定真实成本，不参与搜索。

点击“智能搜索”后，程序根据本地已有 K 线数据自动选择合法的环境周期和入场周期组合。环境周期必须高于入场周期，缺少数据的周期组合直接跳过并显示原因。

## 参数分层

### 第一层：策略结构参数

第一阶段广泛搜索：

- 所有已支持策略。
- 本地数据支持的环境周期 + 入场周期组合。
- 环境回溯窗口：`96 / 192 / 288`。
- 入场回溯窗口：`10 / 20 / 30 / 40 / 50 / 60`。
- 三个基础止盈止损风险轮廓：`(1.0×TP, 1.0×SL)`、`(1.5×TP, 1.0×SL)`、`(1.0×TP, 0.75×SL)`。如果当前 TP 或 SL 为 0，则对应基础值使用单笔逐仓金额的 10%，并继续受现有最大值约束。

杠杆在第一阶段固定为当前值，仓位始终固定。候选组合采用分层抽样，确保每种策略和每个合法周期组合都获得基本覆盖，而不是从完整笛卡尔积中逐项穷举。

### 第二层：局部风险与邻域参数

保留第一阶段前 12 名，并按排名分配不同搜索深度：

- 环境和入场回溯窗口的相邻值。
- 第 1–3 名尝试覆盖全部 10 个杠杆档位。切换杠杆时，止盈止损金额必须按 `new_leverage / base_leverage` 同比缩放，保证价格触发距离不变；缩放后止损超过单笔逐仓金额、止盈超过名义仓位的组合直接排除，禁止截断金额后继续搜索。每个合法杠杆档位搭配一组由固定随机种子选出的相邻回溯和止盈止损参数。
- 第 4–12 名只使用当前杠杆的相邻档位，每名从相邻参数池抽取最多 6 个候选，共最多 54 个候选。
- 止盈和止损分别使用基础值的 `0.75× / 1.0× / 1.25×`，不与所有回溯窗口和杠杆做完整笛卡尔积。

单笔逐仓金额不参与搜索。仓位只会机械放大收益和亏损，不能证明交易信号有效。

### 第三层：稳健性验证

保留第二阶段前 10 名，并执行：

- 样本外时间窗口验证。
- 两个确定性随机窗口验证。
- 对上述验证后的前 3 名，数据足够时再增加 90 天和 180 天窗口验证；数据不足时使用可用的最长窗口并标注。

回测天数不是待优化参数。30 天用于快速初筛，90/180 天用于验证，避免通过选择回测天数故意命中有利行情。

## 搜索预算

- 第一阶段：最多 120 个候选。
- 第二阶段：最多 84 个候选。
- 第三阶段：最多 36 次稳健验证。
- 正常总量：最多约 240 次回测。
- 软截止时间：8 分钟，停止生成新的粗筛或精搜候选。
- 硬截止时间：10 分钟，返回已经完成验证的最佳结果。

程序根据前几次回测的平均耗时动态缩减剩余候选数，优先为第三阶段保留时间。如果时间不足，只返回已完成稳健验证的候选，不把未验证候选伪装成推荐结果。

## 可重复性

分层抽样和随机验证必须使用由搜索输入生成的固定随机种子。同一份数据、同一币种和同一成本配置应得到相同的候选组合与验证窗口，便于复查和比较。

## 排名规则

最终排名优先级：

1. 必须通过现有质量过滤规则。
2. 样本外收益不能明显恶化。
3. 随机窗口通过率和随机最差收益优先。
4. 最大回撤、连续亏损和盈亏比优先于单纯总收益。
5. 只有完成第三阶段的候选可以标记为“稳健”。

## 页面反馈

搜索改成后台任务，避免浏览器持续等待一个 5–10 分钟的 HTTP 请求：

- `POST /api/optimize/jobs` 创建搜索任务并返回 `job_id`。
- `GET /api/optimize/jobs/{job_id}` 返回阶段、进度、耗时、过滤计数和最终结果。
- 前端每秒轮询一次任务状态，任务结束后停止轮询。
- 本地单用户模式同一时间只允许一个搜索任务；进程内最多保留最近 20 个任务，不引入数据库。

搜索过程中显示：

- 当前阶段：粗筛、精搜或稳健验证。
- 已评估数量和总预算。
- 已用时间和预计剩余时间。
- 被质量规则过滤的数量。

结果表新增或保留：

- 环境周期、入场周期和策略。
- 环境/入场回溯窗口。
- 杠杆、止盈和止损。
- 初筛、样本外、随机窗口和长窗口结果。
- 稳健性标签与失败原因。

“套用”操作需要同时写回周期、策略、回溯窗口、杠杆和止盈止损。

## 错误与边界

- 没有任何合法周期组合时停止搜索，提示先拉取哪些本地数据。
- 单个候选失败时记录并跳过，不终止整轮搜索。
- 时间预算耗尽时返回部分结果，并明确标记“提前结束”。
- 搜索结果为空时显示各类过滤原因计数，避免只显示“没有结果”。

## 验证标准

- 搜索不再依赖用户反复手动切换周期、策略和回溯窗口。
- 仓位和回测天数不进入参数优化空间。
- 同一输入重复搜索产生相同候选和验证窗口。
- 模拟慢回测时能在时间预算内缩减候选并返回已验证结果。
- 前后端测试覆盖阶段预算、合法周期组合、固定随机种子、提前结束和套用完整参数。
- 全量 `pytest` 通过，浏览器中能看到阶段进度和完整结果字段。
