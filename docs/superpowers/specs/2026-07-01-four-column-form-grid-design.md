# 回测参数四列布局设计

## 目标

让回测参数区在电脑端稳定显示为每排 4 个控件，避免当前 `auto-fill` 根据窗口宽度产生不确定列数。

## 范围

- 主回测参数和高级成本参数使用相同的响应式列数规则。
- 不修改表单字段顺序、默认值、交互逻辑或后端接口。
- 不调整数据面板、结果指标和表格布局。

## 布局规则

- 宽屏：4 列。
- 中等宽度：2 列。
- 窄屏：1 列。
- 使用 CSS Grid 明确设置列数，不在 HTML 中手工拆分行。

建议断点：

- 默认桌面布局使用 `repeat(4, minmax(0, 1fr))`。
- 视口小于等于 `800px` 时切换为 2 列。
- 视口小于等于 `520px` 时切换为 1 列。

## 实现位置

只修改 `static/css/style.css` 中的 `.form-grid` 和对应媒体查询。`.advanced-grid` 继续复用 `.form-grid`，不增加重复样式。

## 验证

- 添加静态样式测试，确认桌面 4 列以及 2 列、1 列断点存在。
- 运行全部 `pytest` 测试。
- 服务启动后由用户刷新页面，确认当前浏览器宽度下每排显示 4 个控件。
