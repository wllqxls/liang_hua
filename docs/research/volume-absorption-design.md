# 成交量吸收后的短期反转：事件因子设计

## 范围

本研究复用现有 `event_factors.py` 与 `research_event_factors.py`，只增加独立逻辑分支，不创建策略、不接入交易执行，也不删除前三轮失败基线。

- 放宽版同时使用 BTC/USDT、ETH/USDT 的 2024/2025 5m 与 15m 数据。
- 所有年度按真实 UTC 事件时间切片。
- 固定单标的完整往返成本：`2 * (0.0005 + 0.0002) = 0.0014`。
- 5m 与 15m 是预先固定的平行对照，不依据其中一个周期的结果决定是否运行另一个。

## 事件 A：成交量吸收

一根已收盘 K 线同时满足：

1. 当前成交量 / 前 20 根平均成交量 `>= 1.5`，均量不包含当前 K 线；
2. 当前真实波幅 / ATR(14) `<= 1.0`；
3. 当前收盘与 3 根前收盘的位移绝对值 / ATR(14) `>= 1.0`；
4. 位移向上时研究方向为 `SELL`，位移向下时研究方向为 `BUY`。

连续满足条件的行情段只记录首次 A，避免重复样本。

## 事件 B：价格回撤或反弹

A 后检查接下来的 3 根同周期 K 线：

- `SELL`：任一收盘价 `<= A close - 0.5 * A ATR`；
- `BUY`：任一收盘价 `>= A close + 0.5 * A ATR`。

首次满足即记为 B，每个 A 最多一个 B。B 只用于转化率和到达速度统计；A 的固定持有期方向收益用于判断是否存在优势。

## 输出与停止规则

- 5m A 标签：事件后 `5m/15m/1h/4h` 毛收益和扣除 0.14% 后净收益。
- 1h 分组：总体、方向、成交量冲击强度三分位、吸收强度三分位。
- 每组报告样本数、平均毛收益、平均净收益、净胜率、成本后 Profit Factor。
- 生成 2024/2025 的 5m 与 15m 事件 CSV 和同一份对照报告。
- 本轮只允许修改成交量阈值和波幅/ATR 阈值；位移、B、成本、年度切片和分组规则保持不变。
